Serpentine

A bounded, alternating-column track engine that turns a fixed viewport into a continuous snake-shaped queue.

How to pull it

git clone git@github.com:LuckyMonkey/serpentine.git
cd serpentine
npm install
npm run test
npm run typecheck

How to use it

import {
  computeTrackMetrics,
  buildSerpentinePath,
  resolvePlacements
} from "serpentine";

const metrics = computeTrackMetrics({
  width,
  height,
  itemWidth,
  itemHeight,
  gapX,
  gapY,
  padX,
  padY
});

const path = buildSerpentinePath(metrics, itemCount + extraSlots);
const placements = resolvePlacements(path, itemCount, offsetPx);

How to implement it

  1. Measure the viewport that will host the board.
  2. Choose card size, spacing, and padding for that viewport.
  3. Build metrics, then the serpentine path.
  4. Drive an offset from wheel, swipe, keys, or another motion source.
  5. Resolve placements and absolutely position each visible card.

How it works fundamentally

The engine first computes slot rectangles in alternating columns. Even columns run downward; odd columns run upward.

It then turns the slot centers into a single axis-aligned path with cumulative distances. Each item maps to a desired distance on that path.

In tight corner regions the constraint solver nudges later items forward so cards stay readable and collision-free instead of stacking into each other.

How the CSS works

The page CSS is intentionally flat and static. Layout is split into four layers: the top copy block, the metrics row, the bounded stage, and the documentation grid.

The stage itself uses three primitives: an SVG path backdrop, absolutely positioned cards, and a slim progress marker. Cards get their color from a single custom property, --accent.

There is no framework styling dependency on the live page. The root demo is just index.html, demo.css, and demo.js, so GitHub Pages can serve it directly from main.
